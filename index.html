Projects

1) FAQ Assistant with Deterministic Formatting

Goal: answer product FAQs with short, confident responses and a strict JSON output for easy ingestion.

Prompt

Role: You are a precise support assistant.
Context: Answer using the provided knowledge. If missing, say "Insufficient context".
Output: JSON with keys {"answer", "confidence", "sources"}.
Constraints: 2 sentences max for "answer"; confidence is one of [high, medium, low].
Examples:
Input: "Do you ship internationally?"
Output: {"answer":"We currently ship to the US, Canada, and EU.","confidence":"high","sources":["Shipping policy"]}

Success criteria

  • All outputs parse as valid JSON.
  • Answer ≤ 2 sentences; confidence ∈ {high, medium, low}.
  • Mentions “Insufficient context” when the knowledge lacks coverage.

Iteration snapshot

Early attempts returned prose instead of JSON. Adding a concrete example and enumerated constraints fixed structure compliance and reduced retries.

2) Evaluation Rubric for Sales Emails

Goal: score outbound emails on clarity, personalization, and call‑to‑action strength to guide coaching.

Prompt

Task: Score the email on the rubric.
Rubric: clarity (0–5), personalization (0–5), CTA strength (0–5); include 1-line suggestion per category.
Output JSON: {"scores":{"clarity":n,"personalization":n,"cta":n},"suggestions":{"clarity":"...","personalization":"...","cta":"..."},"overall":n}

Success criteria

  • Scores are integers 0–5, overall is average rounded.
  • Each suggestion is ≤ 20 words and actionable.
  • Model never fabricates customer facts.

Iteration snapshot

To stop fabricated facts, the prompt now forbids new claims and requires suggestions framed as edits to the existing text.

3) Data Cleaning Assistant

Goal: inspect a CSV for duplicates, invalid dates, and inconsistent categories; return precise row references.

Prompt

Task: Audit the uploaded CSV for quality issues.
Rules: Refer to rows by the "transaction_id" column only. Propose minimal, reversible fixes.
Output table (Markdown): columns = issue, transaction_id(s), fix.

Success criteria

  • All row references use transaction_id, not row numbers.
  • Proposed fixes are atomic and reversible.
  • Report includes a count of each issue type.

Iteration snapshot

Early drafts mixed row numbers and IDs. A strict rule plus an example table eliminated ambiguity.
